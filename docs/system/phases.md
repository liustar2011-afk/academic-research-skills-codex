# 研究项目阶段定义

本文件定义所有研究项目共用的阶段体系。项目状态不得使用未定义的临时阶段名。

## 阶段总览

| 阶段 | 名称 | 目标 | 典型成果物 | 完成标准 |
|---|---|---|---|---|
| P0 | 项目初始化 | 建立研究项目，明确主题、对象、边界和材料入口 | 项目说明、研究问题、研究边界、材料索引 | 项目目录和基础管理文件已建立 |
| P1 | 材料诊断与框架初建 | 从已有材料中提炼研究问题、初步框架、提纲和模型 | 材料诊断、阶段成果、开题提纲、论文提纲、理论框架、核心模型 | 形成可作为后续补证据的基准稿 |
| P2 | 政策与文献补强 | 补充外部政策、文献、案例证据，支撑 P1 框架 | 政策依据表、文献综述笔记、支线研究成果包 | 关键研究问题和模型有外部依据支撑 |
| P3 | 框架修订与开题成稿 | 根据 P2 证据修订研究问题、框架、提纲并生成开题报告 | 开题报告草稿 v0.1、修订后的提纲和模型 | 开题报告可以进入评审或人工修改 |
| P4 | 论文初稿写作 | 按论文提纲形成章节初稿 | 论文初稿、章节草稿、图表说明 | 主要章节均有初稿 |
| P5 | 审阅与修订 | 根据反馈、证据缺口和逻辑问题修订 | 修改意见、修订记录、版本稿 | 关键问题已处理并记录 |
| P6 | 定稿与输出 | 形成最终论文、报告、PPT 或其他交付物 | 定稿、汇报稿、PPT、摘要 | 交付物完成并标记 final |

## 当前阶段的表达规则

项目状态应写成：

```text
当前处于 P1：材料诊断与框架初建。P1 已完成，下一步进入 P2：政策与文献补强。
```

不要写成未定义的模糊描述，例如：

```text
政策与文献补充前的框架固化阶段
```

除非该阶段已经在本文件中定义。

## 阶段切换规则

每次阶段切换，应同步更新：

1. `PROJECT_INDEX.md`
2. `CURRENT_STATE.md`
3. `TASK_BOARD.md`
4. `CHANGELOG.md`
5. 如有需要，更新 `RESEARCH_DASHBOARD.md`

## 主线与支线关系

支线研究通常发生在 P2、P5，也可在其他阶段临时启动。

- P2 支线：用于补政策、文献、案例和理论依据。
- P5 支线：用于回应评审意见或补强薄弱论证。
