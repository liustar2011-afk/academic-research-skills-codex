## 汇入状态说明

| 状态 | 含义 |
|---|---|
| planned | 计划汇入 |
| partial | 已部分汇入 |
| integrated | 已全部汇入 |
| deferred | 暂缓汇入 |
| rejected | 不汇入 |

## 汇入规则

1. 支线成果不得直接改主线核心文件，除非已有汇入判断。
2. 每次汇入都应说明“吸收了什么、没吸收什么、为什么”。
3. 汇入后同步更新 `PROJECT_INDEX.md`、`TASK_BOARD.md`、`ARTIFACTS.md`、`PROCESS_ARTIFACTS.md` 和 `CHANGELOG.md`。
4. 若汇入影响研究问题、理论框架或论文提纲，应先检查 `DEPENDENCY_MAP.md`。
