# SKILL｜国家能源局能源政策材料口径

## 1. 适用对象

能源政策解读、正式报告、工作方案、请示汇报、PPT脚本、新型电力系统研究、能源保供材料、新能源消纳分析、能源监管方案、绿色低碳转型材料、电力数据场景设计、试点示范推进方案。

## 2. 使用边界

本 Skill 只在“研究成果转化”阶段使用。不得从主题直接生成成稿。

涉及最新政策文件、时间、指标、部门部署和监管要求时，必须查验权威来源。本 Skill 只提供口径和结构，不替代政策事实核验。

## 3. 写作目标

政策口径准确、能源主线清晰、数据支撑充分、监管闭环完整、任务抓手具体、表达正式克制。

生成材料时必须写清：

- 服务什么能源战略和政策部署；
- 面向什么能源领域、业务对象和应用场景；
- 解决什么能源安全、系统运行、消纳、监管、市场或绿色低碳问题；
- 依托什么数据、标准、平台、机制和组织抓手；
- 形成什么监管成果、服务产品、试点案例或运行成效；
- 如何落实安全、合规、责任、进度和闭环评估。

## 4. 标准转化结构

### 能源政策材料

战略要求 → 形势判断 → 现实基础 → 主要矛盾 → 重点任务 → 实施机制 → 监管保障 → 预期成效

### 工作方案

总体要求 → 工作目标 → 重点任务 → 实施路径 → 责任分工 → 进度安排 → 保障措施

### 监管材料

监管依据 → 监管对象 → 监管重点 → 问题发现 → 整改要求 → 跟踪问效 → 长效机制

### 电力数据场景

政策任务 → 业务问题 → 服务对象 → 数据资源 → 分析模型 → 服务产品 → 应用闭环 → 监管/运营价值

## 5. PPT转化规则

- 图形关系体现政策链、业务链、数据链、监管链或项目链；
- 新型电力系统架构页应体现目标、源网荷储协同、调节能力、市场机制、数字化支撑和底座保障；
- 监管闭环页应体现问题发现、台账管理、整改督办、复核销号和长效机制；
- 不把能源政策页写成口号页；
- 不把架构页写成技术堆栈；
- 不使用“不是……而是……”及同类负面对照表达。

## 6. 中电联场景校准

涉及中电联时：

- 中电联可承担政策宣贯、标准支撑、统计监测、专业评价、行业服务和案例总结等行业组织职能；
- 数智公司可支撑平台运营、数据服务、产品开发和项目实施；
- 不得把中电联写成能源监管部门或行政执法主体；
- 涉及数据基础设施时，统一到国家数据基础设施电力行业节点定位。

## 7. 自检清单

- 是否体现能源安全保障、绿色低碳转型、新型电力系统、能源高质量发展等政策导向；
- 是否避免机械复述政策原文；
- 是否转化为行业或单位可执行任务；
- 是否说明运行指标、任务指标或项目指标；
- 是否说明源网荷储、市场机制、监管责任或安全保障关系；
- 是否说明服务对象、数据资源、业务流程、服务产品和应用价值；
- 是否存在空话、套话、宣传腔、技术名词堆砌或同页多职能。
