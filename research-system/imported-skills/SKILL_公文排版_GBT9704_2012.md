# SKILL_公文排版_GBT9704_2012

## 1. 适用场景

用于将 Word 文稿、正式报告、请示、通知、函、纪要、报送稿、内部流转稿等材料，整理为符合或参照 GB/T 9704-2012《党政机关公文格式》的正式公文版式。

本 Skill 只处理版式、格式要素、层级样式和排版自检，不替代行文内容审定、发文程序审核、印章管理和密级管理。

## 2. 使用边界

### 2.1 优先适用

- 党政机关公文格式参照稿
- 央企、行业协会、事业单位正式报送材料
- 请示、报告、通知、函、纪要、说明、方案等正式文稿
- 需要从研究成果资产转化为正式 Word 成稿的材料

### 2.2 谨慎适用

- 对外发布稿、涉密文件、正式红头文件、联合行文文件
- 带印章、签发人、密级、紧急程度、发文字号的正式文件
- 需进入正式流转系统的文件

此类文件须由发文单位按制度复核，本 Skill 只提供排版辅助。

### 2.3 不适用

- PPT 页面排版
- 海报、折页、宣传册
- 学术论文排版
- 合同法律文本专用排版
- 财务报表专用排版

## 3. 核心排版常量

### 3.1 页面规格

| 项目 | 规则 |
|---|---|
| 纸张 | A4，210mm × 297mm |
| 上白边 | 37mm ± 1mm |
| 左白边 / 订口 | 28mm ± 1mm |
| 版心 | 156mm × 225mm |
| Word 实操参考 | 上 3.7cm，左 2.8cm，右 2.6cm，下 3.5cm |
| 文字颜色 | 一般为黑色 |
| 正文排布 | 一般每面 22 行，每行 28 字，撑满版心；特殊情况可适当调整 |

### 3.2 字体字号

| 要素 | 推荐规则 |
|---|---|
| 一般格式要素 | 3号仿宋体 |
| 标题 | 2号小标宋体，居中 |
| 第一层标题 | 3号黑体 |
| 第二层标题 | 3号楷体 |
| 第三、四层标题 | 3号仿宋体 |
| 版记 | 4号仿宋体 |
| 页码 | 4号半角宋体阿拉伯数字 |
| 发文机关标志 | 红色小标宋体，醒目、庄重、美观 |

字体说明：不得随包分发字体文件。若本机缺少小标宋体，应由用户在本机合法安装相应字体，或按单位模板指定字体替代。

## 4. 公文格式要素

公文格式要素按版心内位置划分为：版头、主体、版记。页码位于版心外。

### 4.1 版头

适用于带正式版头的文件。普通报告、汇报稿、研究稿可按单位模板简化。

| 要素 | 编排规则 |
|---|---|
| 份号 | 需要标注时，6位3号阿拉伯数字，顶格排在版心左上角第一行 |
| 密级和保密期限 | 需要标注时，3号黑体，顶格排在版心左上角第二行 |
| 紧急程度 | 需要标注时，3号黑体，顶格排在版心左上角；与份号、密级同现时按顺序分行排列 |
| 发文机关标志 | 居中排布，上边缘至版心上边缘约35mm，红色小标宋体 |
| 发文字号 | 发文机关标志下空二行，居中；年份用六角括号“〔〕”，顺序号不编虚位 |
| 签发人 | 上行文需要时，与发文字号同排；“签发人”用3号仿宋，姓名用3号楷体 |
| 红色分隔线 | 发文字号下约4mm，居中，宽度与版心一致 |

### 4.2 主体

| 要素 | 编排规则 |
|---|---|
| 标题 | 红色分隔线下空二行；2号小标宋；居中；多行标题应词意完整、排列对称、长短适宜 |
| 主送机关 | 标题下空一行，左顶格；回行仍顶格；末尾加全角冒号 |
| 正文 | 主送机关下一行；3号仿宋；每个自然段左空二字，回行顶格；首页必须显示正文 |
| 层级序号 | 依次用“一、”“（一）”“1.”“（1）” |
| 附件说明 | 正文下空一行，左空二字；“附件：”后列附件名称；多个附件用阿拉伯数字标注 |
| 发文机关署名 | 按是否加盖印章、是否联合行文确定位置；普通不加盖印章稿可在正文后右空二字编排 |
| 成文日期 | 用阿拉伯数字写全年、月、日；月、日不编虚位 |
| 附注 | 如有，成文日期下一行，左空二字，加圆括号 |

### 4.3 附件

| 要素 | 编排规则 |
|---|---|
| 附件位置 | 另面编排，位于版记之前，原则上与正文一起装订 |
| 附件标识 | “附件”二字及顺序号用3号黑体，顶格排在版心左上角第一行 |
| 附件标题 | 居中排在版心第三行 |
| 附件格式 | 一般同正文格式 |
| 一致性 | 附件顺序号和标题必须与正文附件说明一致 |

### 4.4 版记

| 要素 | 编排规则 |
|---|---|
| 分隔线 | 与版心等宽；首条和末条用粗线，中间用细线 |
| 抄送机关 | 4号仿宋，左右各空一字；“抄送：”后列机关名称，末尾用句号 |
| 印发机关和印发日期 | 4号仿宋；印发机关左空一字，印发日期右空一字；日期后加“印发” |

### 4.5 页码

| 项目 | 编排规则 |
|---|---|
| 字体 | 4号半角宋体阿拉伯数字 |
| 位置 | 版心下边缘之下，数字左右各放一条一字线 |
| 距离 | 一字线上距版心下边缘约7mm |
| 单双页 | 单页码居右空一字，双页码居左空一字 |
| 附件 | 与正文一起装订时，页码连续编排 |

## 5. Word 成稿执行流程

### 5.1 先判断文种

处理前先判断当前文稿类型：

1. 正式红头文件
2. 报送稿 / 请示 / 报告
3. 函件
4. 纪要
5. 内部研究报告
6. 方案 / 可研 / 说明材料

不同文种决定是否保留版头、发文字号、主送机关、版记、印章区等要素。

### 5.2 再套版式

按以下顺序处理：

1. 页面设置：A4、页边距、版心、页码。
2. 字体样式：标题、正文、层级标题、附件、版记。
3. 段落样式：正文首行缩进二字、回行顶格、标题居中。
4. 格式要素：版头、主送机关、附件说明、署名、日期、版记。
5. 内容校验：附件一致性、层级序号连续性、日期格式、页码连续性。
6. 人工复核：密级、签发人、印章、发文字号、正式发文程序。

### 5.3 Word 样式建议

| 样式名 | 字体 | 字号 | 段落 |
|---|---|---|---|
| 公文标题 | 小标宋体 | 2号 | 居中，多行标题词意完整 |
| 主送机关 | 仿宋 | 3号 | 顶格，末尾全角冒号 |
| 正文 | 仿宋 | 3号 | 首行缩进二字，回行顶格 |
| 一级标题 | 黑体 | 3号 | 首行缩进二字或按单位模板 |
| 二级标题 | 楷体 | 3号 | 首行缩进二字或按单位模板 |
| 三级标题 | 仿宋 | 3号 | 首行缩进二字 |
| 附件说明 | 仿宋 | 3号 | 左空二字 |
| 附件标识 | 黑体 | 3号 | 顶格 |
| 版记 | 仿宋 | 4号 | 左右按版记规则 |
| 页码 | 半角宋体 | 4号 | 按单双页位置编排 |

## 6. 正式材料适配规则

对央企、行业协会、研究报告、可研报告等材料，可采用“公文参照版式”。处理原则：

1. 保留 A4、页边距、标题、正文、层级标题、附件、日期等核心格式。
2. 无正式版头时，不强行添加发文机关标志、红色分隔线、发文字号。
3. 无正式发文流程时，不擅自添加密级、签发人、印章、版记。
4. 报告类材料可保留目录、图表、表格，但正文样式应与公文主体保持协调。
5. 表格过宽时可横排，页码位置应与正文页保持一致。
6. 图表标题、资料来源、注释应使用正式文稿口径，避免说明性自我对话。

## 7. 禁止事项

- 不擅自生成密级、份号、发文字号、签发人、印章。
- 不把“页面职能、写作意图、自检说明、执行器提示”写入正文。
- 不因套格式删除正文中的有效业务信息。
- 不把研究成果直接压缩成空泛公文套话。
- 不在正式稿中使用口语化、营销化、宣传化表达。
- 不使用与正文风格冲突的花哨字体、彩色强调、图标化装饰。
- 不分发字体文件。

## 8. 质量闸门

进入交付前逐项检查：

### 8.1 页面与字体

- [ ] A4 页面设置正确。
- [ ] 页边距符合或参照 37mm / 28mm / 26mm / 35mm。
- [ ] 正文为3号仿宋或单位指定正式字体。
- [ ] 标题为2号小标宋或单位指定标题字体。
- [ ] 正文文字一般为黑色。

### 8.2 结构与要素

- [ ] 标题居中，换行后词意完整。
- [ ] 主送机关顶格，末尾为全角冒号。
- [ ] 正文首行缩进二字，回行顶格。
- [ ] 层级序号使用“一、”“（一）”“1.”“（1）”。
- [ ] 一级、二级、三级标题字体区分正确。
- [ ] 附件说明与附件标题一致。
- [ ] 成文日期用阿拉伯数字，月、日不编虚位。

### 8.3 页码与版记

- [ ] 页码位于版心下边缘之外。
- [ ] 单页码靠右，双页码靠左。
- [ ] 附件与正文一起装订时页码连续。
- [ ] 版记线、抄送机关、印发机关和日期格式一致。

### 8.4 风险复核

- [ ] 密级、份号、紧急程度未被擅自添加。
- [ ] 发文字号和签发人由用户或正式流程提供。
- [ ] 印章区只作占位或由正式文件流程处理。
- [ ] 涉密、发文、盖章事项已标注需人工复核。

## 9. 输出格式

使用本 Skill 处理文稿时，建议输出三类结果：

1. `layout_decision.md`：说明采用何种公文版式、哪些要素保留、哪些要素不添加。
2. `official_doc_style_check.md`：逐项列出格式检查结果和待人工复核事项。
3. `正式稿.docx`：排版后的 Word 文件。
