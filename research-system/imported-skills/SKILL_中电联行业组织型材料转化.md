# SKILL｜中电联行业组织型材料转化

## 1. 适用对象

中电联正式报告、工作方案、请示汇报、PPT脚本、行业统计分析、标准规则材料、会员服务方案、可靠性、信用、国际合作、科技成果转化、数据服务场景设计。

## 2. 使用边界

本 Skill 不作为研究起点，不直接生成正式报告、PPT脚本或工作方案。

调用前必须先完成：

- 研究任务书；
- 决策问题；
- 议题树/假设树；
- 证据池；
- research_asset.md。

本 Skill 仅在“中电联身份校准”和“研究成果转化”阶段调用。

## 3. 核心定位

中电联相关材料应突出：

- 行业组织者；
- 规则支撑者；
- 公共服务者；
- 场景运营组织者。

默认表述：

依托行业组织、标准规则、统计监测、专家资源、会员联系、专业服务和行业数据基础，服务政府决策、支撑行业治理、服务会员企业、促进产业协同，形成兼具公共服务属性和场景运营能力的行业组织型平台。

## 4. 输出结构

### 正式报告

一、背景形势与工作依据  
二、行业需求与现实基础  
三、中电联定位与能力基础  
四、重点任务与场景设计  
五、组织实施与运营机制  
六、风险边界与保障措施  
七、下一步工作建议  

### 工作方案

总体要求 → 工作目标 → 重点任务 → 实施路径 → 组织分工 → 进度安排 → 成果形式 → 保障机制

### 场景设计

场景名称 → 服务对象 → 业务痛点 → 中电联基础 → 数据/规则/专家/会员资源 → 服务内容 → 交付成果 → 运营机制 → 合规边界 → 推广路径

## 5. 中电联承接主线

政策要求 → 行业需求 → 中电联基础 → 场景切口 → 服务产品 → 试点验证 → 行业推广

任何材料都要写清：

- 政策从哪里来；
- 行业缺什么；
- 中电联有什么基础；
- 中电联能组织谁；
- 能形成什么服务；
- 如何试点验证；
- 如何复制推广；
- 如何守住边界。

## 6. 主体分工

| 主体 | 职责 |
|---|---|
| 中电联 | 行业组织、规则牵引、资源协调、公共服务、成果发布 |
| 数智公司 | 技术支撑、平台运营、产品开发、数据服务、项目实施 |
| 专家/机构 | 专业论证、评估评价、合规审查、成果验证 |
| 会员企业 | 需求提出、数据授权、试点应用、反馈迭代 |
| 外部合作方 | 能力互补、资源连接、市场协同 |

## 7. PPT转化规则

- 每页只表达一个核心要点，只承载一个页面职能；
- 页面标题使用短判断句或短任务句；
- 页面中部承载主要信息量；
- 不设置重复标题含义的总结带；
- 底部只放边界、条件、机制或下一步动作；
- 保留原稿有效业务信息，避免关键词清单化；
- 图形体现政策链、业务链、数据链、监管链、服务链或组织链；
- 不写说明性自我表达；
- 不写 OVERLAY 相关内容；
- 字号只设置可读性下限和优先范围，不设置刚性上限。

## 8. 常用表达

优先使用：

- 统筹组织；
- 规则牵引；
- 标准支撑；
- 统计监测；
- 专业评价；
- 会员服务；
- 专家论证；
- 试点验证；
- 成果发布；
- 持续运营；
- 动态更新；
- 闭环管理；
- 授权使用；
- 边界控制。

慎用或禁用：

- 赋能；
- 生态；
- 重塑；
- 颠覆；
- 超级平台；
- 行业大脑；
- 全链路打通；
- 无限价值；
- 本文旨在；
- 本页意在说明；
- 不是……而是……。

## 9. 自检清单

- 中电联行业组织定位是否准确；
- 中电联、数智公司、会员企业、专家机构和外部合作方角色是否清晰；
- 行业需求、资源基础和服务产品关系是否明确；
- 标准、统计、会员、专家、数据等基础支撑是否充分；
- 场景落地和运营机制是否具体；
- 是否保留原稿有效信息；
- 是否存在越位、错位、自我拔高或商业营销化表达；
- 是否通过禁用项检查。
