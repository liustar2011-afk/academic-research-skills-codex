# SKILL｜国家数据局数据要素材料口径

## 1. 适用对象

正式报告、工作方案、政策解读、请示汇报、PPT脚本、场景设计、数据基础设施建设方案、可信数据空间方案、高质量数据集建设方案、试点推进方案、数据产品与运营方案。

## 2. 使用边界

本 Skill 只在“研究成果转化”阶段使用。不得从主题直接生成成稿。

调用前必须确认：

- 已完成研究任务书；
- 已形成议题树/假设树；
- 已建立证据池；
- 已形成 research_asset.md；
- 已明确服务对象、数据资源、场景路径和合规边界。

## 3. 写作目标

政策口径准确、业务主线清晰、场景逻辑完整、任务抓手具体、合规边界明确、语言正式克制。

生成材料时必须写清：

- 服务什么国家战略、政策部署和行业任务；
- 面向什么业务对象和应用场景；
- 使用什么数据资源和数据能力；
- 建立什么机制、流程、标准和平台支撑；
- 形成什么数据产品、服务成果、试点案例或治理成效；
- 如何落实授权、合规、安全、运营和持续迭代。

## 4. 标准转化结构

### 正式方案

政策背景 → 建设定位 → 总体架构 → 重点场景 → 数据与能力支撑 → 运营机制 → 安全合规 → 实施安排

### 场景设计

业务痛点 → 服务对象 → 数据资源 → 数据处理 → 平台能力 → 应用服务 → 交付成果 → 价值成效 → 推广条件

### 数据产品

资源盘点 → 分类分级 → 治理加工 → 产品封装 → 场景应用 → 价值评估 → 持续运营

### 高质量数据集

建设背景 → 应用需求 → 数据来源 → 质量标准 → 标注加工 → 测评机制 → 应用验证 → 持续迭代 → 组织保障

## 5. PPT转化规则

- 页面只表达一个核心要点；
- 标题使用短判断句；
- 主体结构体现业务流、数据流、服务流或治理流；
- 底部只放试点条件、合规边界或下一步动作；
- 不设置与标题重复的总结带；
- 不把页面职能、自检标准、写作意图渲染进页面正文；
- 不使用“不是……而是……”及同类负面对照表达。

## 6. 中电联场景校准

涉及中电联时，必须补充：

- 中电联行业组织、标准规则、会员服务、专家资源、行业数据基础；
- 电力领域数据基础设施作为国家数据基础设施电力行业节点的定位；
- 数智公司技术支撑、平台运营、产品开发和数据服务职责；
- 公共服务、会员增强服务、专题项目服务、数据产品/API服务、生态合作服务边界。

## 7. 自检清单

- 是否避免机械复述政策原文；
- 是否转化为可执行任务；
- 是否写清服务对象、数据来源、平台能力、产品服务和交付成果；
- 是否写清授权、使用范围、审计、责任边界；
- 是否具备试点、复制和持续运营条件；
- 是否存在空话、套话、宣传腔、技术堆砌或说明性文字；
- 是否保留足够信息密度。
