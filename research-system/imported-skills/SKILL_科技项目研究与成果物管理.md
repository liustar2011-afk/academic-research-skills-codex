# SKILL｜科技项目研究与成果物管理

## 0. 定位

本 Skill 用于约束 Codex 或本地工作台管理科技研究项目、研究文档、过程物、交付物、版本和复核记录。它不是研究方法本身，也不替代 ChatGPT 的研究判断。

核心目标：

```text
让项目文档能被打开、被二次修改、被版本化、被复核、被流转，而不是只展示流程。
```

## 1. 管理对象

必须围绕以下对象组织工作：

```text
项目
任务
原始材料
研究过程物
研究成果资产
交付物
版本
修改意见
review_log
artifact_manifest
ChatGPT/Codex handoff
```

不得只围绕“阶段”“按钮”“界面模块”设计流程。

## 2. 文档动作

对任一文档，工作台或 Codex 应支持以下动作：

```text
打开
读取
编辑
保存
另存为 vNext
生成修改提示词
生成审稿提示词
生成转化提示词
生成 Codex 执行任务
追加 review_log
更新 artifact_manifest
归档旧版本
```

## 3. 二次修改规则

每次二次修改必须遵循：

1. 基于原文，不脱离原有有效信息；
2. 保留服务对象、场景、证据、交付、机制、边界、任务等有效信息；
3. 不把高信息密度内容改成关键词清单；
4. 修改后生成新版本，不直接覆盖重要版本；
5. 必须记录修改原因、修改内容、删除内容和仍需核验事项；
6. 涉及正式材料、PPT、Word 时，仍以 research_asset 为母本。

## 4. 版本规则

推荐命名：

```text
主题_文档类型_v01.md
主题_文档类型_v02_根据复核意见修订.md
主题_PPT脚本_v03_强化主链.md
```

若原文件未包含版本号，默认生成：

```text
原文件名_v02.后缀
```

旧版本不得随意删除，应保留或归档至：

```text
11_versions/
99_archive/
```

## 5. review_log 规则

每次版本生成或执行修改后，应追加：

```text
## 时间 + 操作
- 来源文件：
- 新版本：
- 修改原因：
- 保留内容：
- 删除内容：
- 新增内容：
- 仍需核验：
- 下一步建议：
```

## 6. ChatGPT 与 Codex 分工

### ChatGPT 负责

```text
研究判断
材料主线
正式表达
角色边界
政策口径
PPT/Word转化设计
质量复核
```

### Codex 负责

```text
文件创建
文件修改
版本复制
manifest更新
review_log追加
格式检查
打包上下文
生成或执行脚本
```

Codex 不得自行改写研究结论，不得绕过 research_asset 直接生成正式交付物。

## 7. 成果物索引规则

artifact_manifest.md 至少包含：

```text
文件
类型
版本
状态
更新时间
说明
```

当新增、修改、生成版本、归档文件后，必须更新索引。

## 8. 最终标准

合格的科技项目工作台应满足：

1. 文档能打开；
2. 文档能基于原文二次修改；
3. 修改能生成新版本；
4. 修改能进入 review_log；
5. 成果物能进入 artifact_manifest；
6. ChatGPT 任务能从具体文档触发；
7. Codex 执行能被限制在明确文件范围内；
8. 研究资产能作为 PPT/Word/脚本转化母本；
9. 过程物能被复用、流转和归档。
