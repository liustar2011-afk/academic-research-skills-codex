# 研究质量闸门

## 适用场景

Codex 在本地项目中执行文件生成、整理、质检、归档或交付物生成前，应检查研究是否足够进入下一阶段。

## 闸门顺序

1. 研究闸门：材料、证据、任务、成果资产是否齐备。
2. 方法质量：研究问题、假设、证据、结论是否匹配。
3. 交付闸门：是否允许进入 Word/PPT/公文转化。

## 输出

```text
研究闸门结果
方法质量问题
修改文件清单
下一步建议
```

## 基本规则

1. 研究不充分时，创建待补研究清单，不生成交付物。
2. 研究充分时，进入交付转化。
3. 完成后更新 `artifact_manifest.md` 和 `review_log.md`。
