# 支线汇入主线检查清单

支线成果汇入主线前，逐项检查。

## 一、支线成果完整性

- [ ] 是否有 `BRANCH_BRIEF.md`
- [ ] 是否有 `branch_report.md`
- [ ] 是否有 `evidence_table.md`
- [ ] 是否有 `implications_for_main.md`
- [ ] 是否有 `handoff_to_main.md`

## 二、证据质量

- [ ] 政策文件是否记录发布主体、发布时间和来源？
- [ ] 文献是否记录作者、年份、标题、核心观点？
- [ ] 案例是否记录来源、适用范围和局限？
- [ ] 是否区分事实、判断和建议？
- [ ] 是否标注仍需验证的内容？

## 三、主线影响分析

- [ ] 是否说明支撑主线哪个研究问题？
- [ ] 是否影响研究问题？
- [ ] 是否影响理论框架？
- [ ] 是否影响论文提纲？
- [ ] 是否影响开题报告草稿？
- [ ] 是否影响研究边界？

## 四、依赖检查

- [ ] 是否查阅 `DEPENDENCY_MAP.md`？
- [ ] 是否列出需要同步修改的文件？
- [ ] 是否把未完成同步项加入 `REVISION_QUEUE.md`？

## 五、汇入记录

- [ ] 是否更新 `CHANGELOG.md`？
- [ ] 是否更新 `PROJECT_INDEX.md`？
- [ ] 是否更新 `CURRENT_STATE.md`？
- [ ] 是否更新 `07_branch_research/BRANCH_INDEX.md`？

## 六、主线吸收结论

汇入后应明确：

- 已吸收什么。
- 未吸收什么。
- 为什么未吸收。
- 后续还需补什么。
